RK3588 上如果直接运行 rviz2 出现下面这类告警：
libGL error: glx: failed to create dri3 screen
libGL error: failed to load driver: rockchip
原因通常是系统同时安装了 Mali 用户态库和 Mesa，RViz 需要桌面 OpenGL，GLX 在当前显示会话里先尝试了不稳定的 rockchip DRI 路径。
优先使用兼容启动脚本：
cd /home/ztl/Robot-nav/elderbot_ws && bash run_rviz.sh navigation
cd /home/ztl/Robot-nav/elderbot_ws && bash run_rviz.sh slam
如果仍然黑屏、闪退，或者是远程桌面/NX 环境：
cd /home/ztl/Robot-nav/elderbot_ws && bash run_rviz.sh --software navigation
